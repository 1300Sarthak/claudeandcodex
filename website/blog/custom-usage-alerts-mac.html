How to Set Custom Usage Alerts in Claude + Codex Tracker

📅 May 6, 2026 ✍️ Sarthak Sethi ⏱ 4 min read

Claude + Codex Tracker now lets you set completely custom notification thresholds. Instead of the old fixed alerts at 25%, 50%, 75%, and 90%, you can configure up to five percentages that actually matter to your workflow.

Maybe you want a heads-up at 60% so you can pace yourself for a long session. Or maybe you only care about the 85% and 95% marks. This guide shows you how to configure it exactly how you want it.

Why Custom Thresholds Matter

The old fixed system was a blunt tool. Heavy users hitting 75% an hour into a session would get a notification that didn't give them enough runway. Light users would get four notifications per session, most of which they didn't need.

Custom thresholds let you match the alert cadence to how you actually work:

  • Power users: Set a single alert at 90% — you know what you're doing, you just want the final warning.
  • Casual users: Set 50% and 80% — a mid-session check and a clear warning before you get cut off.
  • Researchers: Set 25%, 50%, 75%, 90% — the old default, if that's what you want.
  • Sprint workers: Set a single 95% threshold — you want maximum usage with minimal interruptions.

Setting Up Custom Thresholds

1

Open Settings

Click the Claude + Codex Tracker icon in your menu bar, then click the gear icon (⚙️) in the top-right corner of the popover.

2

Go to Notifications

In the Settings sidebar, click "Notifications". You'll see the current threshold list — by default it shows 75% and 90%.

3

Remove Thresholds You Don't Want

Each threshold row has a red minus button (−) on the right. Click it to remove any threshold you no longer want.

4

Add New Thresholds

Type a percentage in the text field (any number from 1–99) and click "Add". The new threshold appears in the list immediately and is saved automatically.

5

Test It

Hit "Send test notification" at the bottom to confirm macOS delivers the notification correctly. If it doesn't appear, check System Settings → Notifications → Claude + Codex Tracker and make sure alerts are enabled.

Tip: Thresholds apply to both Claude and Codex session usage independently. You'll get a separate notification for each service when either crosses a threshold.

How Many Thresholds Should You Set?

Two thresholds tends to be the sweet spot for most users: a "heads up" alert and a "final warning" alert. More than three and you start getting notification fatigue. For most workflows, we recommend:

  • 70% + 90% — Standard workflow, decent heads up time
  • 85% + 95% — Heavy user who wants maximum usage
  • 50% + 80% — You like to plan and pace your sessions

Notifications Now Use macOS User Notifications

This release also moves from the deprecated NSUserNotification API (deprecated in macOS 11) to the modern UNUserNotificationCenter. This means notifications are more reliable and appear properly in Notification Center, including in Focus modes if you've allowed them.

The first time you enable alerts (or install a new version), you may be asked to grant notification permission. Click "Allow" in the system dialog — the app only uses this to deliver usage alerts.

Thresholds Are Persistent and Per-Session

Your threshold list is saved to disk and persists across app restarts. The notification state (which thresholds have fired) resets when usage drops below a threshold — so if your 5-hour window resets, the alerts will fire again fresh in the next session. No need to manually reset anything.
